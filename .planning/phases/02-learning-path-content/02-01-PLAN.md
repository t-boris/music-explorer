  <files>next.config.ts, src/lib/content.ts, package.json, content/levels/level-0/lesson-1.mdx</files>
  <action>
  Install MDX dependencies: `npm install @next/mdx @mdx-js/loader @mdx-js/react gray-matter remark-gfm rehype-slug rehype-autolink-headings`

  Configure MDX in next.config.ts using @next/mdx with remark-gfm (GitHub-flavored markdown) and rehype-slug + rehype-autolink-headings (linkable headings).

  Create src/lib/content.ts — a content utility module that:
  - Reads MDX files from the `content/` directory at build time
  - Uses gray-matter to parse frontmatter (title, order, levelId, tags, description)
  - Exports functions: `getLevels()`, `getLessons(levelId)`, `getLesson(levelId, lessonSlug)`, `getExercisesForLesson(levelId, lessonSlug)`
  - Uses fs/path for file reading (server-only)
  - Returns typed data matching Level/Lesson interfaces from src/types/index.ts

  Create directory structure:
  ```
  content/
  └── levels/
      └── level-0/
          └── lesson-1.mdx  (minimal test file with frontmatter)
  ```

  The test MDX file should have frontmatter: title, order, levelId, description, tags, and a short body with a heading and paragraph.

  IMPORTANT: Use `import "server-only"` at top of content.ts to prevent client-side import. Theory content is read at request time via server components, NOT bundled into client JS.
  </action>
  <verify>npm run build succeeds with MDX config. Importing content.ts in a server component works.</verify>
