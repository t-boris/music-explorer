  <action>
  **useMetronome hook (src/hooks/use-metronome.ts):**
  "use client" — precise Web Audio API metronome:
  - Create AudioContext on first interaction (browsers require user gesture)
  - Use the "look-ahead scheduler" pattern for precise timing:
    1. Schedule clicks ahead of time using AudioContext.currentTime
    2. Use setInterval (~25ms) to keep scheduling ahead
    3. Schedule OscillatorNode clicks at exact future times
  - Click sound: short sine wave burst (1000Hz for downbeat, 800Hz for other beats, ~50ms duration with quick gain envelope for a clean "tick")
  - State: `{ bpm, isPlaying, currentBeat, beatsPerMeasure }`
  - `setBpm(bpm)`: change tempo (40-240 range)
  - `setBeatsPerMeasure(n)`: change time signature (2, 3, 4, 5, 6, 7)
  - `start()`: begin playback
  - `stop()`: stop playback, reset beat counter
  - `tap()`: tap tempo — calculate BPM from tap intervals (last 4 taps)
  - IMPORTANT: Use the look-ahead pattern, NOT setInterval alone. setInterval has ~10ms jitter which is audible at high BPMs. The AudioContext clock is sample-accurate.
  - Clean up AudioContext on unmount.

  **Metronome component (src/components/practice/metronome.tsx):**
  "use client" — visual metronome UI:
  - Large BPM display in JetBrains Mono (e.g., "120")
  - BPM slider (40-240) with accent color track
  - +/- buttons for fine adjustment (±1 BPM)
  - Increment buttons: +5, +10 for quick jumps
  - Play/Stop toggle button (large, prominent)
  - Beat indicator: row of dots/circles, current beat lights up with accent color
  - Time signature selector (2/4, 3/4, 4/4, 5/4, 6/8, 7/8)
  - Tap tempo button: "TAP" — user taps repeatedly, BPM auto-calculated
  - Visual pulse animation on each beat using Motion (subtle scale on the BPM number)
  - Dark card (surface-800), accent color for active beat and play button
  - Compact enough to embed alongside other practice tools
  </action>

  **Tempo service (src/lib/tempo-service.ts):**
  "use client" — Firestore operations for tempo logging:
  - `logTempoAttempt(userId, data)`: Save to `users/{userId}/tempoAttempts/{id}` with: exerciseId, exerciseTitle, startBpm, targetBpm, achievedBpm, date, createdAt
  - `getTempoHistory(userId, exerciseId)`: Get all attempts for an exercise, ordered by date
  - Add TempoAttempt type to types/index.ts if not already there:
    ```typescript
    interface TempoAttempt {
      id: string; userId: string; exerciseId: string; exerciseTitle: string;
      startBpm: number; targetBpm: number; achievedBpm: number;
      date: string; createdAt: Timestamp;
    }
    ```

  **useTempoTrainer hook (src/hooks/use-tempo-trainer.ts):**
  "use client" — tempo training state machine:
  - State: `{ mode, startBpm, targetBpm, currentBpm, step, isActive }`
  - Modes: "setup" → "training" → "complete"
  - Setup: user sets startBpm, targetBpm, step (BPM increment per round, default 5)
  - Training: metronome plays at currentBpm. User clicks "Got it" to advance by step. User clicks "Struggling" to log current BPM as achievedBpm.
  - Complete: shows summary — started at X, reached Y, target was Z. Option to log attempt.
  - Progression rule: currentBpm starts at startBpm, increments by step each round until targetBpm reached or user stops.

  **TempoTrainer component (src/components/practice/tempo-trainer.tsx):**
  "use client" — tempo training UI:
  - Setup mode: Start BPM input, Target BPM input, Step input (default 5), "Start Training" button
  - Training mode: integrates with Metronome component (plays at currentBpm), shows progress bar (startBpm → targetBpm), "Got it! Next +" button (advance), "That's my limit" button (stop)
  - Complete mode: summary card showing start → achieved → target, "Log Attempt" button, "Try Again" button
  - Progress visualization: horizontal bar showing where currentBpm is between start and target, accent gradient fill
  - BPM history: if user has previous attempts for this exercise, show a small sparkline or list of past achieved BPMs
  - Dark themed throughout
