<task type="auto">
  <name>Task 1: Build audio recorder hook and recording service</name>
  <files>src/hooks/use-audio-recorder.ts, src/lib/recording-service.ts, src/hooks/use-recordings.ts</files>
  <action>
  **useAudioRecorder hook (src/hooks/use-audio-recorder.ts):**
  "use client" custom hook encapsulating MediaRecorder API:
  - `startRecording()`: Request microphone access via `navigator.mediaDevices.getUserMedia({ audio: true })`. Create MediaRecorder with `mimeType: 'audio/webm;codecs=opus'` (fallback to 'audio/webm' if opus not supported). Collect data chunks via ondataavailable.
  - `stopRecording()`: Stop MediaRecorder, assemble Blob from chunks, return Blob + duration.
  - `cancelRecording()`: Stop and discard.
  - State: `{ isRecording, duration, audioBlob, error, permissionStatus }`
  - Duration tracking: start an interval timer on record start, clear on stop.
  - Error handling: microphone permission denied, MediaRecorder not supported.
  - IMPORTANT: Check `MediaRecorder.isTypeSupported()` before setting mimeType. Safari may need 'audio/mp4' fallback.
  - Clean up: stop all tracks on unmount via useEffect cleanup.

  **Recording service (src/lib/recording-service.ts):**
  "use client" — Firestore + Storage operations:
  - `uploadRecording(userId, blob, metadata)`:
    1. Generate unique ID
    2. Upload blob to Firebase Storage at `users/{userId}/recordings/{id}.webm`
    3. Get download URL via `getDownloadURL()`
    4. Create Firestore doc at `users/{userId}/recordings/{id}` with: storageUrl, downloadUrl, duration, createdAt, contextType, contextId, contextTitle, levelId, sessionId
    5. Return the recording doc
  - `getRecordings(userId, contextType?, contextId?)`: Query recordings filtered by context
  - `deleteRecording(userId, recordingId)`: Delete Storage file + Firestore doc
  - Use getFirebaseDb() and getFirebaseStorage() lazy getters.

  **useRecordings hook (src/hooks/use-recordings.ts):**
  "use client" — real-time recording list:
  - `useRecordings(userId, contextType?, contextId?)`: onSnapshot listener on recordings subcollection, optionally filtered by context.
  - Returns { recordings, loading, error }
  </action>
  <verify>TypeScript compiles. Hook types match Recording interface from types/index.ts.</verify>
  <done>MediaRecorder hook handles record/stop/cancel with error handling. Recording service handles Storage upload + Firestore CRUD. Real-time hook for recording list.</done>
