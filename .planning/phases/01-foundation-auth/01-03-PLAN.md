# Plan 01-03: Firestore Schema & Seed Data

**Phase:** 01 — Foundation & Auth
**Depends on:** 01-01 (Firebase config must exist)
**Parallel group:** Can run in parallel with 01-02 after 01-01

## Objective

Deploy Firestore security rules, Storage security rules, and create a seed script that populates the database with the foundational data structure: skill definitions, level stubs (0-12), and a few sample lessons/exercises for level 0. This validates the data model and gives the app real data to render.

## Tasks

### 1. Initialize Firebase project locally
- Run `npm install -g firebase-tools` (if not already installed, gcloud CLI is available)
- Run `firebase init` in project root:
  - Select Firestore (rules + indexes) and Storage (rules)
  - Select existing Firebase project or create new
  - Accept default file paths: `firestore.rules`, `firestore.indexes.json`, `storage.rules`
- This creates `firebase.json`, `.firebaserc`

### 2. Deploy Firestore security rules
- Write `firestore.rules` with the rules from research:
  - Public read for: `levels`, `lessons`, `exercises`, `tests`, `songs`, `skills`
  - No public write for content collections
  - User data: read/write only by owner (`request.auth.uid == userId`)
  - Wildcard subcollection rule for user data
  - Collection group query rule for exercises
- Run `firebase deploy --only firestore:rules`

### 3. Deploy Storage security rules
- Write `storage.rules`:
  - Users can only read/write their own recordings path
  - Content type must be `audio/*`
  - Max file size 50MB
- Run `firebase deploy --only storage`

### 4. Create Firestore indexes config
- `firestore.indexes.json` with composite indexes needed:
  - `users/{uid}/recordings`: `contextType` ASC + `createdAt` DESC
  - `users/{uid}/practiceSessions`: `createdAt` DESC
  - `users/{uid}/testAttempts`: `testId` ASC + `createdAt` DESC
- Run `firebase deploy --only firestore:indexes`

### 5. Create TypeScript seed script
- Create `scripts/seed.ts`:
  - Uses `firebase-admin` SDK (direct Node.js script, not Next.js)
  - Loads service account from env or local key file
  - Seeds in order:
    1. **Skills** (6 docs): rhythm, intervals, chords, fretboard, technique, ear
    2. **Levels** (13 docs): levels 0-12 with order, title, description, skillFocus
    3. **Level 0 lessons** (3-4 docs): Physics of sound lessons
    4. **Level 0 exercises** (2-3 per lesson): Sample exercises
  - Uses batch writes for efficiency
  - Idempotent: uses fixed document IDs so re-running updates rather than duplicates

### 6. Define seed data content
- **Skills:**
  | ID | Name | Description |
  |----|------|-------------|
  | rhythm | Rhythm | Time, tempo, note duration, rhythmic patterns |
  | intervals | Intervals | Distance between notes, recognition, construction |
  | chords | Chords | Chord types, construction, voicings, progressions |
  | fretboard | Fretboard | Note locations, patterns, navigation |
  | technique | Technique | Picking, fretting, coordination, articulation |
  | ear | Ear Training | Pitch recognition, interval identification, chord quality |

- **Levels 0-12:** (titles and descriptions from PROJECT.md curriculum)
  - Level 0: Physics of Sound
  - Level 1: Notation System
  - Level 2: Intervals
  - Level 3: Scales
  - Level 4: Chords
  - Level 5: Functional Harmony
  - Levels 6-12: Roadmap stubs (modes, advanced rhythm, advanced harmony, counterpoint, form, composition, math+music)

- **Level 0 sample lessons:**
  - Lesson 1: "What is Sound?" — frequency, wavelength, amplitude
  - Lesson 2: "Harmonics & Overtones" — harmonic series, timbre
  - Lesson 3: "How We Hear Pitch" — logarithmic perception, octave equivalence

- **Level 0 sample exercises:**
  - "Identify higher/lower pitch" (ear training)
  - "Match the octave" (fretboard + ear)
  - "Clap the rhythm" (rhythm basics)

### 7. Add seed script runner
- Add to `package.json` scripts: `"seed": "npx tsx scripts/seed.ts"`
- Create `scripts/tsconfig.json` for scripts directory (if needed)
- Document in README or `.planning/` how to run seed

### 8. Run seed and verify
- Execute `npm run seed`
- Verify in Firebase Console (or via script output):
  - 6 skill docs created
  - 13 level docs created
  - 3-4 lesson docs under level 0
  - Exercises created under lessons
- Test that the data is readable from the app (optional: quick read in a page)
