# Plan 01-01: Next.js Project Setup, Firebase Config & Base Layout

**Phase:** 01 — Foundation & Auth
**Depends on:** Nothing (first plan)
**Parallel group:** None (must complete before 01-02 and 01-03)

## Objective

Scaffold the Next.js project with Firebase client/admin SDKs, Tailwind v4 dark theme with OKLCH palette, shadcn/ui, fonts, and the base app shell (root layout, top nav, route groups). The result is a running app with the studio aesthetic visible from first load.

## Tasks

### 1. Scaffold Next.js project
- Run `npx create-next-app@latest . --typescript --tailwind --eslint --app --src-dir`
- Verify it runs with `npm run dev`

### 2. Install dependencies
```bash
npm install firebase next-firebase-auth-edge next-themes motion lucide-react react-firebase-hooks server-only
npm install -D firebase-admin
npx shadcn@latest init
```
- Select "New York" style, slate base, CSS variables enabled during shadcn init
- Install initial shadcn components: `npx shadcn@latest add button card avatar dropdown-menu separator`

### 3. Configure environment variables
- Create `.env.local` with all Firebase config keys (placeholders):
  ```
  NEXT_PUBLIC_FIREBASE_API_KEY=
  NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=
  NEXT_PUBLIC_FIREBASE_PROJECT_ID=
  NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=
  NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=
  NEXT_PUBLIC_FIREBASE_APP_ID=
  FIREBASE_PROJECT_ID=
  FIREBASE_CLIENT_EMAIL=
  FIREBASE_PRIVATE_KEY=
  ```
- Create `.env.example` with same keys (no values) for documentation
- Ensure `.env.local` is in `.gitignore`

### 4. Set up Firebase client SDK (singleton)
- Create `src/lib/firebase.ts` — client SDK init with `getApps().length` check
- Export `auth`, `db`, `storage` from single file
- Reference: research code example (exact pattern)

### 5. Set up Firebase Admin SDK (server-only)
- Create `src/lib/firebase-admin.ts` with `import "server-only"` guard
- Singleton init with `cert()` from env vars
- Export `adminAuth`, `adminDb`, `adminStorage`
- Reference: research code example (exact pattern)

### 6. Configure Tailwind v4 OKLCH dark theme
- Replace `globals.css` with OKLCH color tokens using `@theme` directive:
  - surface-900 through surface-600 (dark grays)
  - accent-400 through accent-600 (warm amber)
  - text-primary, text-secondary, text-muted
- Configure shadcn/ui CSS variables to map to OKLCH tokens
- Set dark mode as default

### 7. Set up typography (3 fonts)
- Configure `next/font/google` in root layout:
  - Space Grotesk → `--font-heading`
  - Inter → `--font-body`
  - JetBrains Mono → `--font-mono`
- Apply font variables to `<html>` element
- Set body font-family to Inter by default

### 8. Create root layout with ThemeProvider
- `src/app/layout.tsx`: fonts, ThemeProvider (defaultTheme="dark"), metadata
- `suppressHydrationWarning` on `<html>` to prevent theme flash
- Wrap children with ThemeProvider from next-themes

### 9. Create route group structure
- `src/app/(public)/` — route group for public content (no auth required)
  - `src/app/(public)/levels/page.tsx` — placeholder
  - `src/app/(public)/songs/page.tsx` — placeholder
- `src/app/(auth)/` — route group for auth-gated features
  - `src/app/(auth)/dashboard/page.tsx` — placeholder
  - `src/app/(auth)/practice/page.tsx` — placeholder
  - `src/app/(auth)/progress/page.tsx` — placeholder
- `src/app/login/page.tsx` — placeholder
- Each placeholder: minimal component with page title so routes are navigable

### 10. Build top navigation bar
- Create `src/components/layout/top-nav.tsx`
- Dark background (surface-800), warm accent on active link
- Navigation links: Levels, Songs (public) | Dashboard, Practice, Progress (auth-gated)
- App logo/name "Music Explorer" with Space Grotesk
- Right side: placeholder for user avatar/sign-in button
- Responsive: hamburger menu on mobile (use shadcn Sheet component)
- Add `npx shadcn@latest add sheet` for mobile nav

### 11. Create landing page
- `src/app/page.tsx` — home/landing page
- Hero section with app name, tagline, warm accent gradient
- "Start Learning" CTA button linking to `/levels`
- Dark, immersive aesthetic — first impression of the studio feel

### 12. TypeScript types foundation
- Create `src/types/index.ts` with core entity types:
  - `Level`, `Lesson`, `Exercise`, `Test`
  - `Song`
  - `User`, `Recording`, `TestAttempt`, `PracticeSession`, `ProgressEntry`
  - `Skill`, `ProgressSummary`
- These match the Firestore schema from research

## Verification

- [ ] `npm run dev` starts without errors
- [ ] Home page loads with dark theme, correct fonts, warm accent colors
- [ ] Top nav renders with all links, responsive on mobile
- [ ] All route placeholders accessible via nav
- [ ] No hydration warnings in console
- [ ] Firebase client init doesn't error (even without real credentials)
- [ ] TypeScript compiles with no errors
