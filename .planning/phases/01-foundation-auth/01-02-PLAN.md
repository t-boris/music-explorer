# Plan 01-02: Firebase Auth Integration

**Phase:** 01 — Foundation & Auth
**Depends on:** 01-01 (project setup must exist)
**Parallel group:** Can run in parallel with 01-03 after 01-01

## Objective

Implement Firebase Authentication with Google sign-in, middleware-based route protection using `next-firebase-auth-edge`, auth context for client components, and the login page. Users can sign in with Google, see their avatar in the nav, and protected routes redirect to login.

## Tasks

### 1. Configure next-firebase-auth-edge
- Create `src/lib/auth-config.ts` with shared config:
  - `apiKey` from env
  - `cookieName`, `cookieSignatureKeys`
  - `cookieSerializeOptions` (httpOnly, secure, path, sameSite, maxAge)
  - `serviceAccount` from env vars
- Add cookie secret env vars to `.env.local`:
  ```
  AUTH_COOKIE_NAME=MusicExplorerAuth
  AUTH_COOKIE_SIGNATURE_KEYS=secret1,secret2
  ```
- Update `.env.example`

### 2. Create auth middleware
- Create `src/middleware.ts`:
  - Import `authMiddleware` from `next-firebase-auth-edge`
  - Configure `loginPath: "/api/login"`, `logoutPath: "/api/logout"`
  - `publicPaths`: `["/", "/login", "/levels", "/levels/(.*)", "/songs", "/songs/(.*)"]`
  - `handleInvalidToken`: redirect to `/login` for non-public paths
  - `handleValidToken`: redirect away from `/login` if already authenticated
- Export `config.matcher` excluding static assets and API routes

### 3. Create auth API routes
- `src/app/api/login/route.ts`:
  - GET handler that reads `Authorization: Bearer <idToken>` header
  - Uses `next-firebase-auth-edge` to set auth cookie
  - Returns redirect or JSON success
- `src/app/api/logout/route.ts`:
  - GET handler that clears auth cookie
  - Redirects to home page

### 4. Create auth context provider
- Create `src/components/providers/auth-provider.tsx` ("use client"):
  - React context providing `{ user, loading, signOut }`
  - Uses `onAuthStateChanged` from Firebase client SDK
  - Wraps the app in root layout
- Create `src/hooks/use-auth.ts` — `useAuth()` hook consuming the context

### 5. Build login page
- `src/app/login/page.tsx`:
  - Centered card on dark background
  - "Music Explorer" branding with Space Grotesk
  - "Sign in with Google" button (prominent, warm accent)
  - Subtitle: "Sign in to track your progress"
  - Below: "Or continue browsing" link to `/levels`
  - Uses `signInWithPopup` with `GoogleAuthProvider`
  - After sign-in: POST id token to `/api/login`, then redirect to `/dashboard`
  - Mobile fallback: detect mobile and use `signInWithRedirect`

### 6. Create sign-in button component
- `src/components/auth/sign-in-button.tsx`:
  - Handles Google sign-in flow (popup/redirect)
  - Loading state during auth
  - Error handling with user feedback
  - Reusable (used in login page and optionally in nav)

### 7. Create user menu component
- `src/components/auth/user-menu.tsx`:
  - Shows when authenticated: avatar (Google photo), dropdown with name/email
  - Dropdown items: Dashboard, Settings (placeholder), Sign Out
  - Shows when not authenticated: "Sign In" button/link
  - Uses shadcn DropdownMenu + Avatar components

### 8. Update top nav with auth state
- Modify `src/components/layout/top-nav.tsx`:
  - Right side: UserMenu component (replaces placeholder)
  - Conditionally show auth-gated nav links only when authenticated
  - Public links always visible

### 9. Create user document on first sign-in
- Create `src/lib/user-service.ts`:
  - `ensureUserDocument(uid, displayName, email, photoURL)`:
    - Check if `users/{uid}` doc exists in Firestore
    - If not, create with initial fields (displayName, email, currentLevel: 0, progressSummary: empty, streakDays: 0, totalPracticeMinutes: 0)
    - If exists, update `lastLogin` timestamp
  - Called after successful sign-in (client-side)

### 10. Protect auth-gated pages
- Verify middleware redirects work for:
  - `/dashboard` → redirects to `/login` if not authenticated
  - `/practice` → redirects to `/login` if not authenticated
  - `/progress` → redirects to `/login` if not authenticated
- Verify public pages work without auth:
  - `/`, `/levels`, `/songs` → accessible without sign-in

### 11. Handle sign-out flow
- Sign-out button in UserMenu triggers:
  1. `signOut()` from Firebase client SDK
  2. `fetch("/api/logout")` to clear cookie
  3. Redirect to home page
- Clear auth context state

## Verification
