**2. Create `src/lib/dig-deeper-service.ts`:**

A server-side service that builds prompts and calls the Anthropic API with streaming.

**Types:**

```typescript
interface DigDeeperRequest {
  term?: string;           // Mode 1: the key concept term (e.g., "harmonic series")
  selectedText?: string;   // Mode 2: the user-selected passage
  lessonTitle: string;     // Current lesson title (e.g., "Harmonics & Overtones")
  levelTitle: string;      // Current level title (e.g., "Physics of Sound")
  levelOrder: number;      // Level number (0, 1, 2...) â€” for adjusting explanation depth
}

interface DigDeeperResponse {
  stream: ReadableStream;  // Streaming text response
}
```

**System prompt construction:**

Build a system prompt that:
- Positions Claude as a music theory tutor for an electric guitar student
- References the current lesson and level for context
- Adjusts explanation depth based on `levelOrder` (Level 0 = beginner-friendly, higher = more advanced)
- Instructs Claude to keep responses concise (2-4 paragraphs), use musical examples, and relate to guitar where possible
- Tells Claude to use markdown formatting (bold for key terms, backticks for frequencies/math)

**User prompt:**
- Mode 1 (term): "Explain the concept of '{term}' in the context of {lessonTitle}. What does it mean, why does it matter, and how does it connect to playing guitar?"
- Mode 2 (selectedText): "The student is reading about {lessonTitle} and wants to understand this passage better: \"{selectedText}\". Explain what this means in simpler terms, why it's important, and how it connects to the bigger picture."

**Streaming call:**
- Use `@anthropic-ai/sdk` with `client.messages.stream()` or `client.messages.create({ stream: true })`
- Use model `claude-sonnet-4-5-20250929` (fast, cost-effective for short explanations)
- Set `max_tokens: 1024` (plenty for 2-4 paragraphs)
- Return the SDK's streaming response

**Environment variable:** Read `ANTHROPIC_API_KEY` from `process.env`. Throw a clear error if missing.

**Important:** Mark the file with `import "server-only"` at the top to prevent client-side import.

**Create `src/app/api/dig-deeper/route.ts`:**

A Next.js Route Handler that:

**1. POST handler:**
- Accepts JSON body matching `DigDeeperRequest`
- Validates: must have either `term` or `selectedText` (not both empty), must have `lessonTitle`, `levelTitle`, `levelOrder`
- Returns 400 with descriptive error for invalid requests

**2. Rate limiting (simple in-memory):**
- Use a `Map<string, { count: number; resetAt: number }>` at module scope
- Key by IP address from `request.headers.get("x-forwarded-for")` or `"unknown"`
- Limit: 20 requests per minute per IP
- Return 429 with `Retry-After` header if exceeded
- Clean up expired entries on each request (simple garbage collection)

**3. Streaming response:**
- Call `streamDigDeeperExplanation(body)` from the service
- Convert the Anthropic SDK stream into a standard Web `ReadableStream`
- Use the Anthropic SDK's stream events: listen for `content_block_delta` events and extract `delta.text`
- Return `new Response(readableStream, { headers: { "Content-Type": "text/plain; charset=utf-8", "Cache-Control": "no-cache", "Transfer-Encoding": "chunked" } })`

**4. Error handling:**
- If `ANTHROPIC_API_KEY` is not set, return 503 with "AI service not configured"
- If Anthropic API returns an error, return 502 with "AI service temporarily unavailable"
- Wrap in try/catch, never expose raw API errors to client

**Important:** Do NOT require authentication for this endpoint â€” lessons are public pages. The rate limiting provides abuse protection.
