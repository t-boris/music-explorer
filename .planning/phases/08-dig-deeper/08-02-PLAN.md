**1. Create `src/components/content/dig-deeper-popover.tsx`** — shared popover component:

A "use client" component that displays streaming AI explanations.

Props:
- `isOpen: boolean`
- `onClose: () => void`
- `term?: string` — the concept term (Mode 1)
- `selectedText?: string` — the selected passage (Mode 2)
- `lessonTitle: string`
- `levelTitle: string`
- `levelOrder: number`
- `anchorRect?: DOMRect | null` — positioning anchor (selection rect or button rect)

Behavior:
- When `isOpen` becomes true, call `POST /api/dig-deeper` with the appropriate params using `fetch` with streaming
- Read the response as a `ReadableStream` using `response.body.getReader()` with `TextDecoder`
- Accumulate streamed text chunks into state, re-rendering progressively
- Show 3 states: loading (pulsing dots animation), streaming (text appearing progressively), complete (full text)
- Error state: "Could not load explanation. Please try again." with a retry button
- **Caching:** Use a module-level `Map<string, string>` keyed by `term` or `selectedText` hash. If cached, show immediately without API call.

Positioning:
- Use `position: fixed` with calculated top/left based on `anchorRect`
- Position below the anchor by default, flip above if near viewport bottom
- Max width: 480px, max height: 400px with overflow scroll
- On mobile (viewport width < 640px): position as a bottom sheet (fixed bottom, full width, max-height 60vh)
- Click outside or Escape key closes the popover
- Use `motion` (already installed) for enter/exit animations: fade + slide up

Styling:
- bg-surface-800 border border-surface-600 rounded-xl shadow-2xl
- Header: show the term in bold or "Selected text" label, with a close (X) button
- Body: render streamed text with basic markdown support (bold via regex replace for `**text**` → `<strong>`)
- For full markdown, use a simple approach: split by `\n\n` for paragraphs, replace `**text**` with `<strong>`, replace `` `code` `` with `<code>`
- Footer: subtle "Powered by AI" text in text-muted, small font

**1. Create `src/components/content/text-selection-dig-deeper.tsx`** — Mode 2 floating button:

A "use client" component that detects text selection within a container and shows a floating "Dig Deeper" button.

Props:
- `children: React.ReactNode` — the content area to monitor for selection
- `lessonTitle: string`
- `levelTitle: string`
- `levelOrder: number`

Behavior:
- Listen to `mouseup` and `touchend` events on the container ref
- On event, check `window.getSelection()`:
  - If selection is collapsed (empty) or length < 10 chars, hide button
  - If selection text is valid, get the selection's `getRangeAt(0).getBoundingClientRect()` for positioning
  - Show a small floating button ("Dig Deeper ✨" or with Sparkles icon) positioned near the selection
- On button click: open `DigDeeperPopover` with `selectedText` and the selection rect
- Hide the button when:
  - Selection is cleared (listen to `selectionchange` event on document)
  - Popover is opened (button transforms into popover)
  - User clicks elsewhere
- **Important:** Use `requestAnimationFrame` or small setTimeout (10ms) after mouseup to allow selection to finalize before reading it
- **Important:** Don't show the button if the selection is inside an interactive component (check if selection anchorNode is inside an element with `data-no-dig-deeper` attribute — add this attribute to interactive exercise containers)
- **Mobile:** `touchend` triggers the same logic; the button should be large enough to tap (min 44px touch target)

**Important:** Both `TextSelectionDigDeeper` and `DigDeeperProvider` are client components. Since the lesson page is a server component, create a thin client wrapper component (e.g., `LessonContentWrapper`) that accepts `lesson`/`level` metadata and `children`, and handles the provider + selection wrapping. Pass the `<MDXRemote>` output as children.

Actually, `MDXRemote` from `next-mdx-remote/rsc` renders on the server and produces HTML — so the client wrapper receives the rendered HTML as children. This works fine since client components can receive server-rendered children.

Create a small client component `src/components/content/lesson-content-wrapper.tsx`:
```tsx
"use client";
export function LessonContentWrapper({ lessonTitle, levelTitle, levelOrder, children }) {
  return (
    <DigDeeperProvider lessonTitle={lessonTitle} levelTitle={levelTitle} levelOrder={levelOrder}>
      <TextSelectionDigDeeper lessonTitle={lessonTitle} levelTitle={levelTitle} levelOrder={levelOrder}>
        {children}
      </TextSelectionDigDeeper>
    </DigDeeperProvider>
  );
}
```

Then in the lesson page:
```tsx
<LessonContentWrapper lessonTitle={lesson.title} levelTitle={level.title} levelOrder={level.order}>
  <article className="prose-custom">
    <MDXRemote ... />
  </article>
</LessonContentWrapper>
```
