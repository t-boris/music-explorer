  <action>
Create 6 interactive exercise components in `src/components/exercises/`. All are "use client" components following the pattern established by `src/components/interactive/` (Web Audio API, Canvas/SVG, React state).

Each component accepts `onComplete: () => void` callback that fires when the student successfully completes the exercise. Each also accepts `completed: boolean` to show a "completed" state.

**1. WaveformMatcher** (`waveform-matcher.tsx`) — for ex-0-1-1 "Identify Higher/Lower Pitch"
- Plays two tones using Web Audio OscillatorNode (random frequencies between 200-800 Hz)
- Student selects which is higher/lower via two buttons
- Must get 3 correct in a row to complete
- Shows frequency values after each answer for learning
- Reuse the OscillatorNode + GainNode envelope pattern from frequency-explorer.tsx (10ms attack, 50ms release)

**2. WaveLabeler** (`wave-labeler.tsx`) — for ex-0-1-2 "Describe a Sound Wave"
- Renders an animated sine wave on canvas (reuse pattern from waveform-visualizer.tsx)
- Shows 3 draggable labels: "Frequency", "Amplitude", "Wavelength"
- Student drags each label to the correct position on the wave diagram
- Drop zones: Amplitude = wave height, Wavelength = horizontal cycle length, Frequency = cycles-per-second indicator
- All 3 labels must be correctly placed to complete
- Use HTML drag-and-drop API (onDragStart/onDragOver/onDrop) — no external library needed

**3. HarmonicFinder** (`harmonic-finder.tsx`) — for ex-0-2-1 "Find Natural Harmonics"
- Renders an SVG fretboard (reuse the FretboardDiagram pattern — 6 strings, frets 0-12)
- Student must tap the correct fret positions for harmonics: frets 12, 7, 5
- When a correct position is tapped, play the harmonic frequency using Web Audio (calculate from string fundamental × harmonic number)
- Wrong positions show a brief red highlight; correct ones stay green and play the harmonic tone
- All 3 harmonics (12th, 7th, 5th fret on high E) must be found to complete
- The string fundamental for high E is 329.63 Hz. Harmonic at fret 12 = 2× (659.26 Hz), fret 7 = 3× (988.88 Hz), fret 5 = 4× (1318.51 Hz)

**4. OvertoneCalculator** (`overtone-calculator.tsx`) — for ex-0-2-2 "Harmonic Series Calculation"
- Given fundamental = 110 Hz (A2), student must input the first 4 harmonics (220, 330, 440, 550)
- Shows 4 input fields, one per harmonic (2nd through 5th)
- Each field validates on blur: correct = green border + checkmark, wrong = red border + hint "Multiply fundamental by N"
- All 4 correct to complete
- Simple number inputs — no external deps

**5. OctaveMatcher** (`octave-matcher.tsx`) — for ex-0-3-1 "Match the Octave"
- Renders an SVG fretboard (12 frets, 6 strings)
- Highlights the starting note: A on 5th fret of low E string (110 Hz)
- Student must find the same note one octave up by tapping the fretboard
- Correct positions: 7th fret D string (220 Hz) or 2nd fret G string (220 Hz) or open A string 12th fret (220 Hz)
- When tapped, play the note frequency via Web Audio so student can hear the comparison
- Correct tap → green highlight + play both notes in sequence for comparison → complete

**6. FrequencyCalculator** (`frequency-calculator.tsx`) — for ex-0-3-2 "Equal Temperament Calculation"
- Shows the formula: f_new = 440 × 2^(n/12)
- Presents 4 notes to calculate (C5 = 3 semitones, D5 = 5, E5 = 7, A5 = 12)
- Student enters frequency for each
- Validates within ±0.5 Hz tolerance (accept rounding)
- Show the formula with the specific n value as a hint after wrong answer
- All 4 correct to complete

**Shared patterns:**
- Each component is a self-contained Card with dark theme styling (bg-surface-800, border-surface-700, text-text-primary/secondary)
- Each shows a congratulations animation (green checkmark with motion/react AnimatePresence) on completion
- Each calls `onComplete()` exactly once when all conditions are met
- Use `useRef` for AudioContext (lazy init pattern from existing interactive components)
- Dispose audio resources on unmount

**1. Update Exercise type** in `src/types/index.ts`:
Add an optional `interactiveComponent` field to the `Exercise` interface:
```
interactiveComponent?: "waveform-matcher" | "wave-labeler" | "harmonic-finder" | "overtone-calculator" | "octave-matcher" | "frequency-calculator";
```
This field tells the UI which interactive component to render. Exercises without it keep the current checkbox behavior (backward compatible).

**2. Update MDX frontmatter** for all 3 Level 0 lessons:
Add `interactiveComponent` to each exercise in the frontmatter:
- lesson-1.mdx: ex-0-1-1 → `interactiveComponent: "waveform-matcher"`, ex-0-1-2 → `interactiveComponent: "wave-labeler"`
- lesson-2.mdx: ex-0-2-1 → `interactiveComponent: "harmonic-finder"`, ex-0-2-2 → `interactiveComponent: "overtone-calculator"`
- lesson-3.mdx: ex-0-3-1 → `interactiveComponent: "octave-matcher"`, ex-0-3-2 → `interactiveComponent: "frequency-calculator"`

**3. Update content.ts** to pass through the `interactiveComponent` field from frontmatter (add it to the Exercise mapping in `getExercisesForLesson`).

**4. Refactor ExerciseCard** (`exercise-card.tsx`):
- If `exercise.interactiveComponent` is set and the exercise is NOT yet completed, show a "Start Exercise" button instead of the checkbox
- Clicking "Start Exercise" expands the card to show the interactive component inline (use useState for expanded/collapsed)

- When the interactive component calls `onComplete`, trigger the existing `onToggle` to mark completion in Firestore
- If the exercise IS completed, show a green "Completed" badge and allow re-doing by clicking "Try Again"
