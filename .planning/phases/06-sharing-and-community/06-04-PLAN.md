<task type="auto">
  <name>Task 1: Create comment service</name>
  <files>src/lib/comment-service.ts</files>
  <action>
Create comment-service.ts following the pattern from practice-service.ts:

- `"use client"` directive
- **Collection:** Top-level `comments` collection (not user subcollection — comments come from different users)
- **`addComment(data)`**: Create a comment doc. Takes: authorId, authorDisplayName, authorPhotoURL, targetUserId, targetType, targetId, text. Adds serverTimestamp for createdAt. Returns the new comment ID.
- **`getCommentsForItem(targetUserId, targetType, targetId)`**: Query comments where targetUserId + targetType + targetId match, ordered by createdAt asc. Return Comment[].
- **`getCommentsForUser(targetUserId, limitCount = 50)`**: Query comments where targetUserId matches, ordered by createdAt desc, limit. Return Comment[]. Used for showing all comments on a user's dashboard.
- **`deleteComment(commentId, authorId)`**: Delete a comment doc. Only allow if the current user is the author (check client-side, rules enforce server-side).

**Firestore rules for comments** — Add to firestore.rules:
```
match /comments/{commentId} {
  allow read: if request.auth != null;
  allow create: if request.auth != null && request.resource.data.authorId == request.auth.uid;
  allow delete: if request.auth != null && resource.data.authorId == request.auth.uid;
}
```

**Firestore indexes** — Add composite index for comments:
```json
{
  "collectionGroup": "comments",
  "queryScope": "COLLECTION",
  "fields": [
    { "fieldPath": "targetUserId", "order": "ASCENDING" },
    { "fieldPath": "targetType", "order": "ASCENDING" },
    { "fieldPath": "targetId", "order": "ASCENDING" },
    { "fieldPath": "createdAt", "order": "ASCENDING" }
  ]
}
```

And for user-level comment feed:
```json
{
  "collectionGroup": "comments",
  "queryScope": "COLLECTION",
  "fields": [
    { "fieldPath": "targetUserId", "order": "ASCENDING" },
    { "fieldPath": "createdAt", "order": "DESCENDING" }
  ]
}
```
  </action>

**useComments hook** (`src/hooks/use-comments.ts`):
- `"use client"` directive
- Takes: `targetUserId: string`, `targetType: Comment["targetType"]`, `targetId: string`
- Uses `useAuth()` for current user
- On mount: fetch comments for this item via `getCommentsForItem()`
- State: `comments: Comment[]`, `loading: boolean`, `submitting: boolean`
- `addComment(text: string)`: calls addComment service, optimistically adds to local state
- `deleteComment(commentId: string)`: calls deleteComment service, removes from local state
- Returns `{ comments, loading, submitting, addComment, deleteComment }`

**CommentCard component** (`src/components/community/comment-card.tsx`):
- Props: `comment: Comment`, `canDelete: boolean`, `onDelete?: () => void`
- Displays: avatar (initials from authorDisplayName), author name, relative time, comment text
- If canDelete: show a small trash icon button
- Use existing design tokens: surface-700 bg, text-secondary for metadata, text-primary for content
