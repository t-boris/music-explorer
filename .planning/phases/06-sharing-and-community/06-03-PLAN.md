**useActivityFeed hook** (`src/hooks/use-activity-feed.ts`):
- `"use client"` directive
- Takes `userIds: string[]` (connected user IDs)
- On mount/change: calls `getActivityFeed(userIds)` from activity-service
- Returns `{ events: ActivityEvent[], loading: boolean }`
- When userIds is empty, return empty events immediately

**PeopleList component** (`src/components/community/people-list.tsx`):
- `"use client"` directive
- Props: `connections: Connection[]`
- Renders a horizontal scrollable row (or grid on desktop) of connected people cards
- Each card shows: avatar (initials fallback from displayName), display name, and a link to `/community/{fromUserId}` (the person who shared with me)
- Use the Card component and existing design tokens (surface-800, text-primary, etc.)
- If no connections: show empty state "No connections yet. Ask a friend to share their invite link with you, or share yours from Settings."
- Link to /share for generating invite

**ActivityFeed component** (`src/components/community/activity-feed.tsx`):
- `"use client"` directive
- Props: `events: ActivityEvent[], loading: boolean`
- Renders a chronological list of activity events
- Each event shows: avatar (initials), user name, event description (title field), relative time (e.g., "2 hours ago")
- Use relative time formatting: create a simple `timeAgo(date: Date)` helper inline or in the component
- Event type icons: use lucide icons — CheckCircle for exercise_completed, ClipboardCheck for test_completed, Mic for recording_created, BookOpen for session_logged, ArrowUp for level_up
- If no events: "No recent activity from your connections."
- Loading state: skeleton cards
