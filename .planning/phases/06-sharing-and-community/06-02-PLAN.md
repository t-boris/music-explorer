**Share Settings page** (`/share`) — auth-protected page where users manage sharing:

Create `src/app/(auth)/share/page.tsx` as a `"use client"` page:
- Uses `useAuth()` and `useConnections(user?.uid)`
- Shows the user's invite link (or button to generate one)
- When invite link exists: show the full URL (`/invite/{code}`) with a copy button
- Below: list of "People who can see my dashboard" (connectionsToMe) with remove buttons
- Below: list of "Dashboards I can view" (myConnections) — links to `/community/{userId}`
- Use Card components matching the existing design system

Create `ShareSettings` component in `src/components/community/share-settings.tsx` for the invite card UI:
- Shows invite code prominently
- Copy to clipboard button (use navigator.clipboard.writeText)
- "Your link: {origin}/invite/{code}" display

**Invite acceptance page** (`/invite/[code]`) — auth-protected page for accepting invites:

Create `src/app/(auth)/invite/[code]/page.tsx` as a `"use client"` page:
- Reads `code` from route params
- On mount: calls `acceptInvite(code, user.uid, user.displayName, user.photoURL)`
- Shows loading state while processing
- On success: shows "Connected!" message with link to community tab
- On error: shows error message (invalid/expired code)
- Auto-redirect to `/community` after 3 seconds on success

**Update middleware** — Add `/invite` path pattern to publicPaths if NOT already there. Actually, since invite acceptance requires auth, keep it auth-protected. The middleware already protects all routes not in publicPaths, so `/invite/[code]` is already auth-protected. No middleware changes needed.

However, we need unauthenticated users who click an invite link to be redirected to login first, then back to the invite page. The current middleware redirects to `/login` for unauthenticated users. The login page supports `?redirect=` parameter. Update the invite page to work with this flow:
- When user lands on `/invite/[code]` unauthenticated, middleware sends them to `/login`
- After login, they're redirected back to `/invite/[code]`
- The invite page then processes the acceptance

Check if the middleware passes the redirect parameter. If not, the user just goes to dashboard after login and needs to click the link again — acceptable for MVP.
