  <name>Task 1: Build WaveformVisualizer component</name>
  <files>src/components/interactive/waveform-visualizer.tsx</files>
  <action>
  Create a live waveform visualizer that shows a sound wave in real time, letting students see and hear what they're learning about in Level 0 (Physics of Sound).

  **Component: WaveformVisualizer**
  `"use client"` component using Web Audio API + Canvas:

  **Props:**
  ```typescript
  interface WaveformVisualizerProps {
    defaultFrequency?: number;   // default 440 (A4)
    defaultAmplitude?: number;   // default 0.5 (0-1)
    defaultWaveform?: "sine" | "square" | "sawtooth" | "triangle"; // default "sine"
    showControls?: boolean;      // default true
    width?: number;              // default 600
    height?: number;             // default 200
  }
  ```

  **Features:**
  1. **Canvas waveform display** — draws a real-time oscilloscope view using Web Audio AnalyserNode
  2. **Play/Stop button** — toggles audio output with a sine wave at the set frequency
  3. **Frequency slider** — 20 Hz to 2000 Hz, logarithmic scale. Shows Hz label and note name (e.g., "440 Hz — A4")
  4. **Amplitude slider** — 0 to 1. Changes volume and visually shows amplitude on the waveform
  5. **Waveform type selector** — sine, square, sawtooth, triangle radio buttons
  6. When playing: canvas shows live waveform from AnalyserNode.getByteTimeDomainData()
  7. When stopped: canvas shows a static mathematical rendering of the current waveform

  **Implementation approach:**
  - Create AudioContext lazily on first play (browser requirement)
  - OscillatorNode → GainNode → AnalyserNode → destination
  - Use requestAnimationFrame for canvas drawing loop
  - Clean up AudioContext and animation frame on unmount
  - Use the project's existing OKLCH color tokens: accent-400 for waveform line, surface-800 for background
  - The canvas should have a subtle grid overlay (dark lines) like an oscilloscope

  **Styling:**
  - Dark card with rounded-xl border border-surface-700 bg-surface-800
  - Controls below the canvas in a horizontal row
  - Accent-400 colored waveform line on dark background
  - Compact design that fits within MDX content flow
  </action>
  <verify>Component renders a waveform canvas. Play button produces audio and shows live waveform. Frequency/amplitude sliders work. TypeScript compiles.</verify>
  <done>Interactive waveform visualizer with play/stop, frequency/amplitude/waveform controls, and live oscilloscope display.</done>

  <files>src/components/interactive/interval-player.tsx</files>
  <action>
  Create an interactive interval explorer for Level 2 (Intervals) lessons. Students can click to hear any interval and see it visualized.

  **Component: IntervalPlayer**
  `"use client"` component:

  **Props:**
  ```typescript
  interface IntervalPlayerProps {
    rootNote?: string;     // default "C4"
    showFretboard?: boolean; // default false (simplified version)
  }
  ```

  **Features:**
  1. **Root note selector** — dropdown or piano-key row (C through B)
  2. **Interval buttons** — all 12 intervals from unison to octave:
     - P1 (Unison), m2, M2, m3, M3, P4, TT (Tritone), P5, m6, M6, m7, M7, P8 (Octave)
  3. **Click to hear** — clicking an interval button plays root → interval note sequentially (root for 0.5s, then interval note for 0.5s), then both together for 0.5s
  4. **Visual display:**
     - Show semitone count
     - Show frequency ratio (e.g., 3:2 for perfect fifth)
     - Show consonance/dissonance label (consonant / mild dissonance / strong dissonance)
     - Interval name (e.g., "Perfect Fifth")
  5. **Play modes toggle** — ascending, descending, or harmonic (simultaneous)

  **Implementation:**
  - Use existing `use-audio-generator.ts` patterns for tone generation
  - OscillatorNode with GainNode envelope (10ms attack, 50ms release) — same pattern as ear-training
  - Note-to-frequency conversion using equal temperament: f = 440 * 2^((n-69)/12) where n is MIDI number
  - Interval data as a static array with name, semitones, ratio, consonance level

  **Styling:**
  - Interval buttons in a horizontal wrapping row
  - Active/playing interval highlighted with accent-400 ring
  - Consonance labeled with color: green (consonant), yellow (mild), red (strong dissonance)
  - Dark card container
  </action>
