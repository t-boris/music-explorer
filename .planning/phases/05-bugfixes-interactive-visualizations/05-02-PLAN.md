  Fix the navigation bar flash when auth state is resolving. Currently, when the page loads:
  1. `loading=true, user=null` → only public links shown
  2. Auth resolves → `loading=false, user={...}` → auth links suddenly appear, causing a layout shift

  **Fix approach:**
  During the loading state, show the auth links section as a skeleton placeholder to prevent layout shift:
  - When `loading` is true, render placeholder spans with the same width as the auth link labels (approximate) using `animate-pulse bg-surface-700 rounded` divs
  - This reserves the space and prevents the jump
  - On mobile, during loading, hide the auth links entirely (the hamburger menu handles them)

  **Alternative simpler fix:**
  If the user was previously signed in (check for existence of auth cookie via a quick flag), show auth links immediately and let them correct on auth resolve. This avoids the flash in the common case.

  Use the simpler approach: just check `loading` and render skeleton placeholders for the auth links area on desktop:
  ```tsx
  {loading && (
    <div className="hidden md:flex items-center gap-6">
      <Separator orientation="vertical" className="h-5" />
      {[1, 2, 3].map((i) => (
        <div key={i} className="h-4 w-16 animate-pulse rounded bg-surface-700" />
      ))}
    </div>
  )}
  ```

  Also apply to mobile: show skeleton items in the Sheet when loading.
  </action>

  <action>
  Add proper page metadata (title, description) to pages that currently lack it.

  **Challenge:** Dashboard, practice, and progress are `"use client"` pages. Client components cannot export `generateMetadata`. Two options:
  1. Create a `layout.tsx` in `src/app/(auth)/` with metadata for the group
  2. Add individual metadata via layout files per route

  **Recommended approach:** Create `src/app/(auth)/layout.tsx` that wraps all auth pages with a metadata definition:
  ```tsx
  import type { Metadata } from "next";

  export const metadata: Metadata = {
    robots: "noindex", // Auth pages shouldn't be indexed
  };

  export default function AuthLayout({ children }: { children: React.ReactNode }) {
    return <>{children}</>;
  }
  ```

  Then create per-route metadata via adding a `layout.tsx` or using a different Next.js pattern. Since these are client pages, the cleanest approach is to use `<title>` in the `<head>` via the `metadata` export from a server layout or by adding a thin server layout wrapper per page.

  **Simplest approach — individual layout.tsx per route:**
  - `src/app/(auth)/dashboard/layout.tsx` — metadata: "Dashboard | Music Explorer"
  - `src/app/(auth)/practice/layout.tsx` — metadata: "Practice | Music Explorer"
  - `src/app/(auth)/progress/layout.tsx` — metadata: "Progress | Music Explorer"

  Each layout is a minimal pass-through that just exports metadata:
  ```tsx
  import type { Metadata } from "next";
  export const metadata: Metadata = {
    title: "Dashboard | Music Explorer",
    description: "Your personal practice dashboard.",
  };
  export default function Layout({ children }: { children: React.ReactNode }) {
    return <>{children}</>;
  }
  ```

  **Login page:** Already a client component. Add a layout.tsx at `src/app/login/layout.tsx`:

    description: "Sign in to track your guitar learning progress.",
  };
  export default function Layout({ children }: { children: React.ReactNode }) {
